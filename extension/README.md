# 蝦皮備貨天數一鍵改 1 天（Chrome 擴充功能）

每周自動把蝦皮台灣賣場所有商品的**備貨天數改為 1 天**。
完整流程：登入 → 更新商品資料 → 下載備貨天數模板（zip，內含多個 xlsx）→
把每個 xlsx 的備貨天數欄改 1 → 逐個上傳。

## 安裝

1. 開啟 Chrome → `chrome://extensions`
2. 右上角開啟「開發人員模式」
3. 「載入未封裝項目」→ 選本目錄 `extension/`
4. 工具列釘選本擴充功能

## 設定

點擴充功能圖示，在彈窗填入：

- **蝦皮帳號** / **密碼**：登入備援用
- **SPC_F cookie**：優先用它免登入（從蝦皮瀏覽器 cookie 複製 `SPC_F` 的值）

點「儲存設定」。

## 使用

點「▶ 立即執行」。擴充功能會：
1. 注入 SPC_F cookie，開啟蝦皮賣家中心（必要時用帳密登入）
2. 進到「更新商品資料 → 下載」，勾選備貨天數，點下載產生模板
3. 輪詢直到檔案產出，下載 zip
4. 解壓、把每個 xlsx 的備貨天數欄全部改 1
5. 切到上傳 tab，**逐個上傳**（每個等處理完成再傳下一個）
6. 完成通知

彈窗下方即時顯示記錄。

## 每周自動

安裝時自動建立每周鬧鐘（7 天一次）。到期自動觸發上述流程。
（條件：瀏覽器需在背景執行；MV3 service worker 會在觸發時喚醒。）

## 結構

| 檔案 | 用途 |
|---|---|
| `manifest.json` | MV3 設定 |
| `config.js` | 共用常數（URL、selector、欄位） |
| `background.js` | service worker：注入 cookie、抓下載、解壓改值（JSZip）、鬧鐘、記錄 |
| `content.js` | 頁內自動化狀態機：登入/生成/輪詢/下載/逐個上傳 |
| `popup.html` / `popup.js` | 設定 + 一鍵執行 + 記錄 |
| `lib/jszip.min.js` | 瀏覽器內解壓 / 改 xlsx |
| `icons/icon128.png` | 圖示 |

## 流程細節（實測結論）

- **備貨天數欄**：xlsx 第 1 列 key `et_title_product_dts`（第 I 欄）；前 4 列為表頭，第 5 列起為資料（每列一個商品變體）。
- **下載**：點「下載」產生 → 輪詢 `get_mass_record_list` 直到 `record_status==1` → 點該列下載拿 zip。
- **上傳**：**傳 xlsx，不是 zip**（傳 zip 會「格式錯誤」）；**一次只能處理一份**，必須逐個上傳、等前一個「完成」再傳下一個。
- **499/500**：正常，少數商品本身問題失敗；可下載失敗明細（只列失敗項）。本工具會記錄但不中斷。
- openpyxl/xlsx 相容性坑（`activePane="bottom_left"`）以 XML 層級修改規避——本擴充用 JSZip 直接改 sheet XML，保留格式。

## 已知限制 / 待觀察

- 登入偶爾會卡驗證碼/簡訊（此時需人工在瀏覽器完成，再重跑）。
- 蝦皮改版時 selector 可能失效，集中在 `config.js` 修正即可。
